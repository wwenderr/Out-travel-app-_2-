# OutTravel — концепция iOS-приложения для трекинга путешествий

## 1) Цель приложения
OutTravel помогает отмечать посещённые страны и города, смотреть статистику поездок, видеть свои отметки на карте и собирать личные гайды по городам (места + фото + описание + координаты).

---

## 2) MVP (первая рабочая версия)

### Основные экраны
1. **Статистика**
   - Количество посещённых стран.
   - Количество посещённых городов.
   - Количество поездок за год.
   - Топ стран/городов по количеству посещений.

2. **Карта**
   - Карта мира с пинами посещённых городов.
   - Кластеризация пинов на дальнем масштабе.
   - Фильтр: все поездки / по году / по стране.

3. **Каталог городов**
   - Список городов, в которых есть поездки.
   - Поиск и фильтры.
   - Переход в карточку города.

4. **Гайды по городу**
   - Для каждого города пользователь создаёт места для посещения.
   - Карточка места: название, описание, фото, адрес/координаты, категория.
   - Карта внутри города с отметками мест.

---

## 3) Архитектура iOS

### Рекомендуемый стек
- **UI**: SwiftUI
- **Архитектура**: MVVM
- **Хранилище**: SwiftData (или Core Data)
- **Карты**: MapKit
- **Фото**: PhotosPicker + локальное хранение/CloudKit
- **Синхронизация (этап 2)**: CloudKit

### Модули
- `Trips` — поездки и посещения.
- `Stats` — агрегаты и аналитика.
- `Map` — карта с пинами и фильтрами.
- `Guides` — места, фото, заметки.
- `Settings` — экспорт/импорт, язык, тема.

---

## 4) Модель данных (MVP)

### Country
- `id: UUID`
- `name: String`
- `isoCode: String`

### City
- `id: UUID`
- `name: String`
- `countryId: UUID`
- `latitude: Double`
- `longitude: Double`

### Trip
- `id: UUID`
- `cityId: UUID`
- `startDate: Date`
- `endDate: Date`
- `notes: String?`

### Place (точка в гайде)
- `id: UUID`
- `cityId: UUID`
- `title: String`
- `description: String`
- `category: String` (еда, вид, музей, прогулка и т.д.)
- `latitude: Double`
- `longitude: Double`
- `photoURLs: [String]`
- `rating: Int?`

---

## 5) Пользовательские сценарии

1. Добавить поездку:
   - Выбрать страну/город.
   - Указать даты.
   - Сохранить.
   - Приложение автоматически обновляет статистику и карту.

2. Добавить место в гайд города:
   - Открыть город.
   - Нажать «Добавить место».
   - Ввести описание, добавить фото, поставить точку на карте.

3. Просмотр статистики:
   - За всё время.
   - За конкретный год.
   - По странам/городам.

---

## 6) Вычисляемая статистика

- Уникальные посещённые страны.
- Уникальные посещённые города.
- Всего поездок.
- Поездки по месяцам (график).
- Топ-5 стран по количеству поездок.
- Средняя длительность поездки.

Формула для длительности поездки:
`durationDays = max(1, endDate - startDate в днях)`

---

## 7) UX-детали

- Онбординг с 3 шагами: «добавь первую поездку» → «отметь на карте» → «создай первое место в гайде».
- Быстрая кнопка `+` для добавления поездки или места.
- Оффлайн-режим: все базовые функции работают без интернета.
- Экспорт в JSON/CSV (этап 2).

---

## 8) Этапы разработки

### Этап 1 (2–4 недели)
- Базовые сущности (страны, города, поездки).
- Экран статистики.
- Карта посещённых городов.
- Каталог городов.

### Этап 2 (2–3 недели)
- Гайды: места, категории, фото.
- Карта мест внутри города.
- Поиск и фильтры.

### Этап 3 (1–2 недели)
- Синхронизация CloudKit.
- Экспорт/импорт.
- Улучшенная аналитика.

---

## 9) Риски и решения

- **Дубликаты городов** → нормализация по `name + country + координаты`.
- **Большое число пинов на карте** → кластеризация и lazy-loading.
- **Рост объёма фото** → сжатие + фоновые загрузки.

---

## 10) Что можно сделать прямо сейчас

1. Создать SwiftUI-проект `OutTravel`.
2. Добавить модели `Country`, `City`, `Trip`, `Place`.
3. Сделать 4 таба: `Stats`, `Map`, `Cities`, `Guides`.
4. Реализовать «Добавить поездку» и «Добавить место».
5. Привязать вычисление статистики к хранилищу.

Этот документ можно использовать как техническое ТЗ для MVP.
